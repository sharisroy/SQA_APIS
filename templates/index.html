Signup API

Endpoint: POST /auth/signup

Headers: Content-Type: application/json

Request Body

{
    "name": "John Doe",
    "email": "john@example.com",
    "password": "your_password"
}

Response Body

{
    "success": true,
    "code": 200,
    "message": "User registered successfully.",
    "user": {
        "email": "john@example.com",
        "name": "John Doe"
    }
}

Login API

Endpoint: POST /auth/login

Headers: Content-Type: application/json

Request Body

{
    "email": "john@example.com",
    "password": "your_password"
}

Response Body

{
    "success": true,
    "code": 200,
    "message": "Login successful.",
    "user": {
        "email": "john@example.com",
        "name": "John Doe"
    },
    "auth": {
        "access_token": "xxxxxx",
        "expires_at": "2025-10-09T15:30:00+06:00"
    }
}

Profile API

Endpoint: GET /auth/profile

Headers: Authorization: Bearer <access_token>

Query Params

Optional: none

Response Body

{
    "success": true,
    "code": 200,
    "user": {
        "email": "john@example.com",
        "name": "John Doe",
        "created_at": "2025-01-01T12:00:00+06:00"
    }
}

Notes API

Headers: Authorization: Bearer <access_token>, Content-Type: application/json

3️⃣ Create Note

Endpoint: POST /note

Request Body

{
    "title": "New Note",
    "note": "This is the content of the new note"
}

Response Body

{
    "success": true,
    "code": 201,
    "message": "Note created successfully",
    "note": {
        "id": "note_46",
        "title": "New Note",
        "note": "This is the content of the new note",
        "created_at": "2025-10-09T10:00:00+06:00"
    }
}

1️⃣ Get Notes

Endpoint: GET /note

Query Parameters (Optional)

{
    "page": 1,
    "limit": 10,
    "search": "keyword"
}

Request Body

Not required

Response Body

{
    "success": true,
    "code": 200,
    "page": 1,
    "limit": 10,
    "total": 45,
    "notes": [
        {"id": "note_1", "title": "First Note", "note": "This is the first note", "created_at": "2025-10-08T12:00:00+06:00"},
        {"id": "note_2", "title": "Second Note", "note": "This is the second note", "created_at": "2025-10-08T12:05:00+06:00"}
    ]
}

2️⃣ Get Note by ID

Endpoint: GET /note/<note_id>

Request Body

Not required

Response Body

{
    "success": true,
    "code": 200,
    "note": {
        "id": "note_1",
        "title": "First Note",
        "note": "This is the first note",
        "created_at": "2025-10-08T12:00:00+06:00",
        "updated_at": null
    }
}

4️⃣ Update Note

Endpoint: PATCH /note/<note_id>

Request Body

{
    "title": "Updated Note Title",
    "note": "Updated content of the note"
}

Response Body

{
    "success": true,
    "code": 200,
    "message": "Note updated successfully",
    "note": {
        "id": "note_46",
        "title": "Updated Note Title",
        "note": "Updated content of the note",
        "updated_at": "2025-10-09T10:15:00+06:00"
    }
}

5️⃣ Delete Note

Endpoint: DELETE /note/<note_id>

Request Body

Not required

Response Body

{
    "success": true,
    "code": 200,
    "message": "Note deleted successfully",
    "note_id": "note_46"
}

Note: All note endpoints require a valid Authorization header. Pagination and search are optional for listing notes.
