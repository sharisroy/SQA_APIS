Welcome to SQA API Documentation

This documentation covers all endpoints for Authentication and Note Management. Use the sidebar to navigate through API sections.

All APIs require Bearer Token in Authorization header (except signup and login). The API follows REST principles and returns JSON responses.

Signup

POST /auth/signup

Register a new user account.

Request Body 
{
  "name": "John Doe",
  "email": "john@example.com",
  "password": "password123"
}
Response Body 
{
  "success": true,
  "code": 200,
  "message": "User registered successfully.",
  "user": {
    "email": "john@example.com",
    "name": "John Doe"
  }
}

List Notes

GET /note/?page=1&per_page=10

Retrieve paginated list of user's notes.

Headers 
Authorization: Bearer <access_token>
Query Parameters 
page: 1
per_page: 10
Response Body 
{
  "code": 200,
  "message": "All notes retrieved successfully",
